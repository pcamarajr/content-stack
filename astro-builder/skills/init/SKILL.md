Interactive setup wizard for astro-builder. Configures the plugin for your Astro project through a guided conversation.

**Arguments:** $ARGUMENTS

---

## Step 0 — Route

Parse $ARGUMENTS:

- **`project`** → Read `skills/init/rounds/project.md` (relative to plugin root) and follow its instructions completely
- **`lighthouse`** → Read `skills/init/rounds/lighthouse.md` (relative to plugin root) and follow its instructions completely
- **Empty** → Run the Status Dashboard (below)
- **Unknown argument** → Tell the user the valid options: `project`, `lighthouse`

After completing any round, always end with a short "What's next?" line pointing to the next incomplete round.

---

## Status Dashboard

Only runs when $ARGUMENTS is empty.

### 1. Detect project basics

Try to read `package.json` → extract `name`.
Try to read `CLAUDE.md` → check if it was generated by astro-builder (look for `.astro-builder/` references).

### 2. Determine round completion

| Round | Complete when |
|-------|---------------|
| project | `CLAUDE.md` exists AND all four files exist: `.astro-builder/style-guide.md`, `.astro-builder/content-schema.md`, `.astro-builder/design-system.md`, `.astro-builder/anti-patterns.md` |
| lighthouse | `.astro-builder/lighthouse.json` exists AND `.husky/pre-push` exists |

Use 🔄 (started) when some but not all required files for a round exist.

### 3. Output

```text
## astro-builder — Setup Status

Site:   [name from package.json, or "—"]
Config: .astro-builder/ ([found | not found])

✅ /astro-builder:init project    — [one-line summary, e.g. "CLAUDE.md + 4 config files generated"]
⬜ /astro-builder:init lighthouse — not started

→ Next: /astro-builder:init [first incomplete round]

What each step does:
  project    — Scan repo, interview, generate CLAUDE.md + .astro-builder/ config files
  lighthouse — Set up automated Lighthouse CI auditing on git push
```

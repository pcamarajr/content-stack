Interactive setup wizard for astro-builder. Configures the plugin for your Astro project through a guided conversation.

**Arguments:** $ARGUMENTS

---

## Step 0 ‚Äî Route

Parse $ARGUMENTS:

- **`project`** ‚Üí Read `skills/init/rounds/project.md` (relative to plugin root) and follow its instructions completely
- **`lighthouse`** ‚Üí Read `skills/init/rounds/lighthouse.md` (relative to plugin root) and follow its instructions completely
- **Empty** ‚Üí Run the Status Dashboard (below)
- **Unknown argument** ‚Üí Tell the user the valid options: `project`, `lighthouse`

After completing any round, always end with a short "What's next?" line pointing to the next incomplete round.

---

## Status Dashboard

Only runs when $ARGUMENTS is empty.

### 1. Detect project basics

Try to read `package.json` ‚Üí extract `name`.
Try to read `CLAUDE.md` ‚Üí check if it was generated by astro-builder (look for `.astro-builder/` references).

### 2. Determine round completion

| Round | Complete when |
|-------|---------------|
| project | `CLAUDE.md` exists AND all five guidance files exist: `.astro-builder/style-guide.md`, `.astro-builder/content-schema.md`, `.astro-builder/design-system.md`, `.astro-builder/anti-patterns.md`, `src/styles/global.css` ‚Äî AND the code scaffold exists: `astro.config.ts`, `tsconfig.json`, `src/content.config.ts`, `src/layouts/BaseLayout.astro`, all four of `src/lib/i18n.ts` / `src/lib/urls.ts` / `src/lib/format.ts` / `src/lib/content.ts`, at least one `src/i18n/*.json`, `public/robots.txt` |
| lighthouse | `.astro-builder/lighthouse.json` exists AND `.astro-builder/pre-push.sh` exists AND `.husky/pre-push` exists |

Use üîÑ (started) when some but not all required files for a round exist.
